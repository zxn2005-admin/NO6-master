# 就诊通知系统修改说明

## 概述
本次修改完善了医院管理系统的就诊通知功能，实现了以下需求：

1. 用户预约成功后立即发送所有后续提醒，而非分时段发送
2. 判断用户接收状态，通知发送失败有系统记录和重试机制
3. 管理员可以在后台对系统记录进行查看与处理

## 功能说明

### 1. 预约成功后立即创建通知
- 当用户预约成功时，系统会立即创建3种类型的通知：
  - 预约确认
  - 就诊前1天提醒
  - 就诊前1小时提醒

### 2. 通知发送和重试机制
- 定时任务每30秒检查一次待发送的通知
- 通知发送失败会自动重试，最多重试3次
- 超过3次失败的通知会标记为"失败"状态

### 3. 管理员后台管理
- 管理员可以通过 `/tongzhirecord/page` 接口查看所有通知记录
- 可以查看通知状态（待发送、发送中、成功、失败、待重试）
- 可以通过 `/tongzhirecord/retry/{id}` 接口手动重试失败的通知

### 4. 通知状态
- 待发送：通知已创建，等待发送
- 发送中：通知正在发送
- 成功：通知发送成功
- 失败：通知发送失败且超过重试次数
- 待重试：通知发送失败，等待下次重试

## API接口

### 通知记录管理
- `GET /tongzhirecord/page` - 分页查询通知记录
- `GET /tongzhirecord/info/{id}` - 获取通知记录详情
- `POST /tongzhirecord/retry/{id}` - 手动重试发送通知
- `POST /tongzhirecord/delete` - 删除通知记录

## 注意事项

1. 需要先执行数据库更新脚本
2. 通知发送逻辑在 `NotificationScheduler.simulateSendNotification()` 方法中，目前是模拟成功，实际使用时需要替换为真实的短信/邮件发送代码
3. 定时任务每30秒执行一次，可以根据需要调整
